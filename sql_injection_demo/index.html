Part (a) — Vulnerable Form UNSAFE — SQL Injection

User input is concatenated directly into the SQL query string. A crafted input can alter the query's logic and retrieve data without knowing the correct values.

Injection payload to try: In the Brand field, type: ' OR 1=1--
Leave Model and Colorway blank and click Search (Unsafe).
The -- starts a SQL comment that discards the rest of the WHERE clause, turning it into WHERE Brand = '' OR 1=1 — always TRUE — returning every sneaker in the database without knowing any real brand name.

Part (b) — Prepared Statement SAFE — Injection Prevented

The same query is rewritten using parameterized placeholders (?). SQLite binds the values separately — injected SQL is treated as plain text and cannot alter the query's structure.

Try the same payload here: Enter ' OR 1=1-- in the Brand field and click Search (Safe).
The prepared statement treats the entire string as a literal value, so the database looks for a brand literally named ' OR 1=1-- — no rows match, injection fails.
