Database Project Description
Introduction:
You will have the opportunity to investigate and gain practical experience in database design and
programming through a project. The project involves designing a database and providing comprehensive
supporting evidence of its sound design, as well as developing the associated front-end and back-end
components to implement common functionalities such as record insertion, updating, deletion, browsing,
searching, and filtering.
The complexity, difficulty, scope, and theme of a team project may vary considerably. The only
requirement for this project is that it must be related to the course material covered. MySQL should be
used to create and query the database, while PHP or any other tool with which you are proficient may be
used to develop an interface that allows users and administrators to access and modify the database.

Team Structure:
The project is a team-based assignment designed to help students develop skills for becoming effective
collaborators in a software project. Each team will consist of four to six students, with one student
designated as the team leader. All team members are expected to contribute to every project deliverable.
Important Note: To successfully complete the database project for this course, you must first become familiar
with installing the required software and learning how to connect HTML to a MySQL database using PHP. A
variety of online tutorials and instructional videos are available to guide you through this process.

Phase 2 (20%): (Due on Friday 03/13/2026)
Our next phase is designing the database. All you need to do for this part is to provide an ER (or EER)
model for your database and then convert it into relations (tables).

Task A: Draw an ER (or EER) diagram.
The ER (or EER) diagram should accurately represent the proposal you have provided. If new entities are
needed, you should add (or edit) them after starting with the same relations that you described in your
proposal (in Phase I). The entities, their characteristics, and their connections (relationships) must all be
identified.
For each entity (for each relation), provide total participation, and (min, max) constraint.
There are many utilities on the web for drawing these ER (or EER) diagrams. For example
https://creately. com/ and www.draw.io.

Task B: Convert your ER diagram into relations.
Once the ER (or EER) diagram is ready, you need to convert the ER (or EER) model into a relational
database schema.
Your report of this phase should have the following components:
1. The ER (or EER) diagram.
2. A list of relations and their attributes.
• Indicate the primary key and foreign keys (if any) for each relation.
• Write a detailed description of each attribute (for each table), its purpose, and data type.
• Indicate each attribute’s default value (if any) or if the attribute can be set to ‘null’.
• Explain the actions taken on any foreign key when the primary key (that the foreign key refers to) is deleted
(For example no action, delete cascade, set null, or set default).
2. A mapping from the ER (or EER) diagram to the Relation Database Schema.

How to submit:

submit this pdf file via eLearning.
Phase 3 (30%): (Due on Sunday 04/05/2026):
In the first part of this phase, you are required to provide a list of all the relations you are using, along

In the second part of this phase, you will primarily work with HTML, PHP, and SQL. For the first time,

Task A: Third Normal Form Normalization
You must ensure that all relations for this project are in Third Normal Form (3NF). If any table is not in
3NF, it must be normalized accordingly. After the decomposition process, all resulting relations should be
in 3NF.
Create a file named TaskA.pdf that documents the conversion of the initial schema into the final schema,
with all relations expressed in Third Normal Form (3NF). This file should also include all the functional
dependencies identified at the start.
Task B: Create forms using HTML.
In Phase I, you presented your web interface as graphics. This time, you are required to deliver the
interface as HTML files. Provide a list of all HTML files included in your interface. These interfaces
are primarily intended to capture user input; database connectivity and PHP code are not required for
this task. The focus is solely on the interface. If necessary, you may also include JavaScript files for
input validation and CSS files for styling.
• For this stage, CSS and JavaScript are not required.
• Provide HTML files that will be filled with the results of queries. There is no need to connect to the
databases to present results for this task; only note that you will do so later.
Create a file called taskB.pdf and add all the HTML, CSS, and JavaScript (if any) files to it.
Task C: Creating and loading relations.
The relations (tables) required for your project were created in Phase 2. In this phase, you will define the
actual tables. To create all tables in your database, prepare a file named create.sql. The data files for these
tables should be either tab- or comma-delimited (csv files or data files in any other format). You may
generate these data files yourself using dummy values. Additionally, create a load.sql file that includes
all the data files.
Create a file named TaskC.pdf and include the create.sql and load.sql files.
